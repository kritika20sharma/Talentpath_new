1. Core Framework & Language
Tool / Technology Version Role Purpose & How It Is Used
Next.js
v16.1.4
 16.1.4 Full-stack web framework The core framework powering the entire website. Handles all page routing (Home, About, Industries, Services, Jobs, Blogs, Contact), server-side rendering for SEO, image optimisation, font loading, and API capabilities. Built using the modern App Router architecture.
React
v19.2.3
 19.2.3 UI component library Powers every interactive element — navigation bar, hero section, contact form, job cards, industry pages, and more — as reusable, composable components. React is the industry-standard library for building modern user interfaces.
TypeScript
v5.x
 5.x Programming language A superset of JavaScript that adds strict type checking to all code. Reduces bugs at development time, makes the codebase easier to maintain, and ensures better long-term reliability. 100% of the website code is written in TypeScript.
Node.js
v22.x
 22.x JavaScript runtime The server-side runtime environment required to build and run the Next.js application. Used during development and by the build pipeline. Not exposed to website visitors directly.
2. Styling & Animation
Tool / Technology Version Role Purpose & How It Is Used
Tailwind CSS
v4.x
 4.x (via PostCSS) CSS styling framework Handles all visual design — layout, spacing, typography, colour, responsive breakpoints, hover effects, and dark mode readiness. All styling is applied as utility classes directly in component files. No separate custom CSS files are needed for standard UI.
Framer Motion
v12.39.x
 12.39.0 Animation library Powers all entrance animations (fade-in, slide-up), hover lift effects on cards, and staggered content reveals on the hero section, about page, and service cards. Provides a premium, polished feel to the UI without impacting performance.
Poppins (Google Fonts)
—
 Loaded via CDN Primary typeface The Poppins typeface is used for all headings, body text, buttons, and UI elements across the website. Loaded using Next.js's built-in Google Fonts integration (weights 400–900), which ensures fonts are self-hosted for performance and privacy.
3. Content Management System (CMS)
Tool / Technology Version Role Purpose & How It Is Used
Sanity CMS
v5.23.x
 5.23.0 Headless content management The content management platform used to create, edit, and publish all dynamic website content — including blog posts and job listings. Authorised team members log in to the Sanity Studio (a web-based editor accessible at /studio on the website domain) to manage content without touching code. Content is delivered to the website via Sanity's API.
next-sanity
v12.3.x
 12.3.1 CMS integration layer The official connector between Sanity CMS and Next.js. Handles fetching content from the Sanity API, embedding the Sanity Studio within the website, and enabling live content previews during editing.
4. Forms & Communication
Tool / Technology Version Role Purpose & How It Is Used
Web3Forms
API v1
 API v1 Contact form delivery service Handles all contact form submissions from the website's Contact Us page. When a visitor submits the form (name, email, phone, message, enquiry type, and optional CV attachment), Web3Forms securely receives the data and forwards it to career@talentpath.co.in. No backend server or database is required — the form communicates directly with the Web3Forms API. Includes built-in spam protection via a honeypot field.
5. Media & Asset Management
Tool / Technology Version Role Purpose & How It Is Used
Next.js Image
Built-in
 Built into Next.js Image optimisation All images on the website (team photos, industry banners, logo, founder photos) are served through Next.js's built-in Image component. It automatically converts images to modern WebP format, resizes them to the exact dimensions needed per screen size, and lazy-loads them so the page loads faster. No manual image editing is required.
FFmpeg
v8.x
 8.x Video processing tool Used as a one-time processing tool to merge and normalise multiple background video clips (meeting room, office, and team footage) into a single optimised video file (combined.mp4) at 1920×1080 resolution and 30 frames per second. This combined video is used as the looping background on the website's hero section. FFmpeg is a developer tool and is not part of the live website.
Google Maps
—
 Linked (no API key) Office location link The "Visit Us" contact card links to a Google Maps search for the TalentPath office at Procapitus Business Park, Sector 63, Noida. This is a standard URL link — no Google Maps API key or embed is used.

8. Deployment & Hosting (Recommended)

Vercel (Hosting Platform)

The recommended production hosting platform for Next.js applications, built by the same team that created Next.js. Connects directly to the GitHub repository — every time code is pushed to the main branch, the site automatically re-deploys within ~60 seconds. Includes global CDN, automatic SSL, and 99.99% uptime SLA. Free tier available; Pro plan recommended for production traffic.

Custom Domain (talentpath.co.in)

The production website domain is configured by adding a CNAME DNS record pointing to Vercel's servers. SSL/HTTPS certificates are automatically provisioned and renewed at no cost. Domain registration and DNS management are handled separately through the domain registrar (e.g., GoDaddy, BigRock, or Hostinger).

Important Notes for Client Records
  • Web3Forms account: The contact form is linked to a Web3Forms account registered with career@talentpath.co.in. All form submissions will be delivered to this inbox. The Web3Forms access key must be kept secure and not shared publicly.
  • Sanity CMS project: Blog posts and job listings are managed through the Sanity Studio. Team members who need editorial access must be added as users in the Sanity project settings at sanity.io/manage.
  • GitHub repository access: The source code is stored at github.com/kritika20sharma/Talentpath_new. Access is controlled via GitHub account permissions. Any developer working on the site in future must be granted repository access.
  • No databases used: The website does not use any traditional SQL or NoSQL database. Content is managed via Sanity CMS and form submissions are handled via Web3Forms — no self-hosted database infrastructure is needed.
  • No backend server: The website runs entirely on Next.js (serverless) deployed on Vercel. There is no separate Node, PHP, or Python server to maintain.
  • Open-source libraries: All frameworks and libraries listed in this document are open-source and commercially free to use (MIT / Apache 2.0 licensed), with the exception of Sanity CMS and Vercel which have free tiers and paid plans for higher usage.
